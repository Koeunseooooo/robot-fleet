# 세션 킥오프 (오늘 할 일 제안)

fleet_web은 학습 목적 사이드 프로젝트라, 매 세션 시작 시 사람이 직접 "오늘 뭐 하지"를 정하지 않고 `docs/WBS.md`를 기준으로 다음 작업을 제안받는다. `scripts/next-wbs-task.sh`가 SessionStart 훅으로 이미 다음 Day를 알려주지만(대화 맨 앞에 `WBS 알림 (fleet_web): ...` 형태로 표시됨), 이 스킬은 그 알림을 실제 "오늘의 작업 지시"로 구체화하는 절차다.

## 절차

1. `docs/WBS.md`에서 상태가 `⬜ 예정`인 가장 앞선 Day를 확인한다 (SessionStart 알림에 이미 나와 있으면 그걸 사용, 없으면 `scripts/next-wbs-task.sh`를 직접 실행하거나 `docs/WBS.md`를 Read).
2. `docs/FEATURES.md`에서 그 Day에 대응하는 기능(F 번호)을 찾아 상세 스펙(설명/완료기준/예외)을 같이 확인한다.
3. 사용자에게 "오늘은 Day N: <작업 내용>을 진행하면 됩니다" 형태로 제안하고, 완료 기준과 관련 스펙을 짧게 요약해서 보여준다.
4. 사용자가 그 작업에 대해 질문하거나(오늘 작업 관련이든, 프로젝트 전반에 대한 것이든) 세부 작업을 더 쪼개고 싶어 하면 같이 논의하며 실제 구현으로 이어간다. 이 단계는 정해진 스크립트가 없다 — 자연스러운 대화로 진행한다.
5. 하루에 여러 Day를 진행하고 싶다고 하면, 다음 `⬜ 예정` Day도 이어서 확인해준다.
6. 작업이 끝나면 `docs/WBS.md`의 해당 Day 상태를 `✅ 완료`로 갱신할지 확인하고, [[obsidian-log]] → [[ship]] 순서로 마무리한다 (자세한 흐름은 `docs/ROUTINE.md` 참고).

## 주의사항

- WBS의 다음 Day를 무조건 강요하지 않는다 — 사용자가 다른 걸 하고 싶어 하면 그쪽을 우선한다. 이 스킬은 "결정 피로를 없애는 기본값"이지 강제 순서가 아니다.
- 모든 Day가 `✅ 완료`면 (`scripts/next-wbs-task.sh`가 "남은 작업 없음"을 알려줌) 다음 Phase 추가나 새로운 백로그 항목을 사용자와 상의한다.
