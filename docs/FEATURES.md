# 기능 명세서 (Feature Specification)

[`WBS.md`](./WBS.md)가 "언제 무엇을 하는지"를 정하는 문서라면, 이 문서는 **각 기능이 정확히 무엇을 하는지**(입출력, 상태, 예외, 완료 기준)를 정의한다. WBS의 Day 번호를 기준으로 대응시켰다.

작업 중 실제 구현이 스펙과 달라지면(불가피하게 자주 있는 일이다), 이 문서를 먼저 고치고 코드를 맞춘다 — 스펙이 뒤늦게 코드를 따라가지 않도록 한다.

---

## 공통 데이터 모델

Phase 1~3에서 공유하는 타입. `src/types`에 그대로 옮겨 구현한다 (WBS Day 6).

```ts
type RobotStatus = "Running" | "Error" | "Charging";

interface Position {
  x: number; // 격자 좌표계, 0 이상
  y: number;
}

interface Robot {
  id: string;          // "robot-1" 형태
  name: string;         // "1호 로봇" 등 화면 표시용
  position: Position;
  status: RobotStatus;
  updatedAt: string;    // ISO 8601
}

type FleetEventType = "collision" | "obstacle";

interface FleetEvent {
  id: string;
  type: FleetEventType;
  robotId: string;
  message: string;      // "3호 로봇 충돌 감지" 등 사람이 읽을 메시지
  createdAt: string;    // ISO 8601
}
```

### Mock 서버 인터페이스

| 단계 | 프로토콜 | 엔드포인트/메시지 | 설명 |
| --- | --- | --- | --- |
| Phase 1 (Polling) | REST | `GET /robots` | 현재 로봇 3~5대의 스냅샷(`Robot[]`) 반환 |
| Phase 1 (WebSocket 전환 후) | WS | `{ type: "robots:update", payload: Robot[] }` | 0.5초 주기로 전체 스냅샷 push |
| Phase 3 | WS (서버→클라이언트) | `{ type: "event", payload: FleetEvent }` | 무작위 충돌/장애물 이벤트 push |
| Phase 3 | WS (클라이언트→서버) | `{ type: "command:move", payload: { robotId: string; target: Position } }` | 원격 이동 명령 전송 |

## Phase 1 — 2D 격자 지도와 실시간 뽈뽈이

### F1-1. Mock 서버: 좌표/상태 시뮬레이션 (Day 2~5)
- **설명**: 3~5대 로봇을 인메모리에 유지하며 0.5초마다 좌표를 소폭 랜덤 이동시키고, 낮은 확률로 상태를 `Running/Error/Charging` 중 하나로 전환한다.
- **완료 기준**: `GET /robots`를 여러 번 호출하면 매번 좌표가 조금씩 달라짐. 상태값도 시간이 지나면 바뀜.
- **예외**: 좌표는 격자 범위(예: 0~19) 밖으로 나가지 않도록 clamp.

### F1-2. 로봇 좌표/상태 구독 (Polling → WebSocket) (Day 8, 16~17)
- **설명**: FE는 초기엔 TanStack Query `refetchInterval: 500`으로 `GET /robots`를 폴링하고, 이후 WebSocket으로 전환해 서버 push를 받는다. 어느 쪽이든 최종적으로 `robotStore`(Zustand)를 갱신한다.
- **완료 기준**: 네트워크 탭/콘솔에서 0.5초 주기 갱신 확인. Polling→WebSocket 전환 후에도 화면 동작은 동일.
- **예외**: WebSocket 연결 끊김 시 자동 재연결(간단한 지수 백오프면 충분).

### F1-3. 격자 배경 지도 (Day 9)
- **설명**: OpenLayers `Vector Layer` 또는 커스텀 타일 없이, 단색 배경 위에 일정 간격의 격자 선을 그린 레이어.
- **완료 기준**: 지도를 확대/축소해도 격자가 깨지지 않고 유지.

### F1-4. 로봇 마커 렌더링 + 이동 애니메이션 (Day 10~11)
- **설명**: `robotStore`의 좌표를 OpenLayers `Vector Layer` 포인트로 그리고, 좌표가 바뀔 때 즉시 점프하지 않고 보간(tween)으로 부드럽게 이동.
- **완료 기준**: 좌표 갱신 시 마커가 순간이동 없이 미끄러지듯 이동.
- **예외**: 로봇이 목록에서 사라지면(연결 끊김 등) 마커도 제거.

### F1-5. 좌측 로봇 리스트 (Day 12~13)
- **설명**: 로봇 ID/이름/상태를 리스트로 표시. 상태별 색상 배지(`Running`=초록, `Error`=빨강, `Charging`=노랑 등). 리스트 항목 클릭 시 해당 로봇을 "선택 상태"로 만들고 지도가 포커스.
- **완료 기준**: 리스트 클릭 → 지도 위 해당 로봇 강조(예: 링 표시) 및 중심 이동.

### F1-6. 이동 궤적 표시 (Day 14)
- **설명**: 선택된 로봇의 최근 N개 좌표(예: 최근 30개)를 이어 `LineString`으로 그린다.
- **완료 기준**: 로봇 클릭 후 이동할수록 궤적이 늘어나며 화면에 남는다. 다른 로봇 선택 시 이전 궤적은 사라짐.
- **예외**: 궤적 포인트는 무한히 쌓이지 않도록 최대 개수 제한.

### F1-7. Error 상태 인터랙션 (Day 15)
- **설명**: 로봇 상태가 `Error`로 바뀌면 지도 위 마커와 리스트 항목 모두 빨간색으로 깜빡인다(CSS/스타일 애니메이션).
- **완료 기준**: 상태가 `Error`가 아닐 때는 깜빡임이 없고, `Error`가 되는 즉시 시작, 벗어나면 즉시 멈춤.

## Phase 2 — 3D 뷰어 전환

### F2-1. 3D 기본 씬 (Day 20~22)
- **설명**: `Canvas` + 바닥(`PlaneGeometry`) + 로봇당 큐브(`BoxGeometry`) 하나. 로봇 좌표(x, y)를 3D 좌표(x, z)로 매핑하고 `useFrame`에서 `robotStore`를 구독해 위치를 갱신.
- **완료 기준**: 2D에서 로봇이 움직이면 3D 큐브도 동일한 타이밍으로 움직임 (같은 데이터 소스 공유 확인용).

### F2-2. 카메라 컨트롤 (Day 23)
- **설명**: `@react-three/drei`의 `OrbitControls`로 드래그 회전/줌.
- **완료 기준**: 마우스 드래그로 360도 회전, 휠로 줌 인/아웃.

### F2-3. 2D/3D 토글 (Day 24~25)
- **설명**: 화면 상단 토글 버튼으로 2D 지도 컴포넌트 ↔ 3D 씬 컴포넌트를 전환. 상태(`viewMode: "2d" | "3d"`)는 Zustand에 둔다.
- **완료 기준**: 토글 전환 시 데이터 유실 없이(선택된 로봇 등 상태 유지) 뷰만 바뀜.

### F2-4. 로봇 카메라 추적 (Day 26~27)
- **설명**: 로봇 더블클릭 시 카메라가 해당 로봇 뒤(3인칭)로 부드럽게(lerp) 이동해 계속 따라간다. 다른 로봇 더블클릭 시 대상 전환, 배경 더블클릭 시 추적 해제.
- **완료 기준**: 추적 중 로봇이 이동해도 카메라가 끊김 없이 따라감. 전환 시 급격한 점프 없음.

## Phase 3 — 관제 이벤트 및 제어

### F3-1. 이벤트 발행/구독 (Day 30~31)
- **설명**: Mock 서버가 무작위 주기로(예: 10~30초) 임의 로봇에 대해 `FleetEvent`(충돌/장애물)를 생성해 WebSocket으로 push. FE는 이벤트 구독 훅으로 수신해 Zustand 이벤트 큐에 추가.
- **완료 기준**: 콘솔에서 이벤트 수신 로그 확인, 큐에 누적.

### F3-2. 토스트 알림 (Day 32)
- **설명**: 이벤트 수신 시 화면 우상단 등에 토스트 표시(로봇 이름 + 메시지), 몇 초 후 자동 소멸.
- **완료 기준**: 이벤트 발생 시 토스트가 뜨고, 여러 개 겹치면 쌓여서 표시.

### F3-3. 경고 마커 (Day 33)
- **설명**: 이벤트 대상 로봇의 2D/3D 위치에 ⚠️ 아이콘을 일정 시간(예: 5초) 표시.
- **완료 기준**: 2D/3D 어느 뷰에 있든 이벤트 발생 로봇 위치에 마커가 보임.

### F3-4. 원격 이동 명령 (Day 34~37)
- **설명**: 사용자가 지도 위 빈 좌표를 클릭하면 "선택된 목표 좌표"로 표시되고, `[원격 이동 명령]` 버튼을 누르면 `command:move` 메시지를 WebSocket으로 전송. Mock 서버는 수신 즉시 해당 로봇의 목표 좌표를 갱신하고, 이후 시뮬레이션 루프에서 그 좌표로 서서히 이동시킨다.
- **완료 기준**: 명령 전송 후 로봇이 실제로 지정 좌표 방향으로 이동을 시작해 도착.
- **예외**: 로봇이 이미 다른 목적지로 이동 중이면 최신 명령으로 덮어쓴다. 존재하지 않는 로봇 ID로 명령이 오면 무시.
