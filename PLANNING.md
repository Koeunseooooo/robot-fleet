# Mini-Fleet: Light-weight Robot Control System

## 프로젝트 개요

로봇관제 팀 합류를 목표로 진행하는 개인 사이드 프로젝트. 무거운 3D 창고 시뮬레이션을 처음부터 만드는 대신, **2D 지도 위 실시간 로봇 위치 매핑**을 먼저 검증하고, 이후 3D 뷰어와 제어(Control) 기능까지 단계적으로 확장한다.

- **중심축**: Next.js 기반 프론트엔드
- **핵심 가치**: 실시간 데이터 시각화 + 인터랙션 + 관제/제어 스토리라인
- **진행 방식**: Phase 1 → 2 → 3 순서로, 각 Phase는 독립적으로 데모 가능한 상태를 목표로 함

## 기술 스택 (예정)

| 영역 | 기술 |
| --- | --- |
| 프레임워크 | Next.js |
| 2D 지도 | OpenLayers |
| 3D 렌더링 | Three.js (React Three Fiber) |
| 실시간 통신 | Polling → WebSocket 전환 |
| Mock 서버 | 별도 mock 서버 (좌표/상태/이벤트 발행) |
| 상태관리 | (Phase 진행하며 필요 시 결정 — Zustand 등 검토) |

---

## 📍 Phase 1 — MVP: 2D 격자 지도와 실시간 뽈뽈이 (기반 다지기)

**목표**: OpenLayers와 Mock 데이터를 활용해 실시간 위치 매핑 기본기 완성.

**화면 구성**
- 좌측: 로봇 리스트 (상태 표시)
- 우측: 2D 지도 영역

**핵심 기능**
- **지도(OpenLayers)**: 오픈스트리트맵 대신 창고 도면 느낌의 단색 격자(Grid) 배경 레이어
- **데이터 처리**: Mock 서버로부터 3~5대 가상 로봇의 좌표 `(x, y)` + 상태(`Running` / `Error` / `Charging`)를 0.5초 주기로 수신 (초기: Polling → 추후: WebSocket 전환)
- **시각화**: `Vector Layer` 위 로봇 아이콘(또는 점)이 끊김 없이 부드럽게 이동(Animation)

**재미 요소**
- 로봇 클릭 시 이동 궤적(Line) 지도에 표시
- 상태가 `Error`로 전환되면 빨간색 깜빡임 인터랙션

**완료 기준**: 로봇 리스트 ↔ 지도 연동, 실시간 이동, 궤적/에러 인터랙션 동작

---

## 🪵 Phase 2 — 3D 뷰어 전환: Three.js 맛보기 (흥미 유발)

**목표**: Phase 1에서 검증된 좌표 데이터를 Three.js(React Three Fiber) 기반 3D 공간으로 확장.

**핵심 기능**
- 화면 상단 **[2D 도면 보기] / [3D 관제 보기] 전환 토글 버튼**
- **3D 공간**: `Canvas` + 바닥(`PlaneGeometry`) + 큐브 로봇(`BoxGeometry`)
- Phase 1 실시간 좌표 데이터를 `useFrame` 훅에서 반영해 큐브 이동

**재미 요소**
- 마우스 드래그로 360도 회전
- 특정 로봇 더블클릭 시 카메라가 해당 로봇을 추적(3인칭/1인칭 뷰)

**완료 기준**: 2D/3D 토글 정상 동작, 동일 좌표 데이터가 3D에서도 동기화되어 이동, 카메라 추적 기능 동작

---

## 🚨 Phase 3 — 관제 이벤트 및 제어 (스토리라인 완성)

**목표**: 단순 모니터링을 넘어 '제어(Control)' 개념 도입으로 직무 연관성 확보.

**핵심 기능**
- **이벤트 발생**: Mock 서버가 무작위로 충돌/장애물 감지 이벤트를 WebSocket으로 발행 (예: "3호 로봇 충돌 감지")
- **알림 UI**: 토스트 알림 + 2D/3D 화면 내 해당 로봇 위치에 ⚠️ 경고 마커 표시
- **웹 제어(Command)**: 지도 위 빈 좌표 클릭 → `[원격 이동 명령]` 실행 → 서버로 좌표 Publish → 로봇이 해당 좌표로 이동 시작

**완료 기준**: 이벤트 알림 end-to-end 동작, 사용자 명령이 실제 로봇 이동으로 반영

---

## 진행 메모

- 각 Phase 완료 시 데모 가능한 상태로 만들고 다음 Phase 진입
- Mock 서버 스펙(좌표 포맷, 이벤트 포맷 등)은 Phase 1 착수 시 확정
